Decapsulates the key that is contained by the ciphertext.

This process is based on the BB1-KEm as proposed in the IEEE P1363.3 draft standard. The key is associated with the given identity.

Parameters
key[out] the decapsulated key
cipher[in] ciphertext that encapsulates the key
id[in] the id of the party that the cipher is destined for

Decapsulates the key that is contained by the ciphertext.

Takes the identity's private key and the received ciphertext to compute the session key to be used. Note that the current implementation does not use the identity string, but always takes the same private key.

Parameters
keythe resulting session key
cipherthe ciphertext to be decrypted
idthe identity string used
